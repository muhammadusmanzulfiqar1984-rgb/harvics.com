3Proposed unified architecture

Base: Project A (harvyXb2b-main) — keep its App-router UI, shadcn/Radix kit, LLM router, agents/ML, Prisma schema, Apollo/SendGrid/Twilio wiring.

UINext 16 App router · shadcn / Radix · Zustand · TanStack Query

DataPrisma 7 + Postgres · 17 models (from A) · JSON fallback dev
ChannelsSendGrid · Twilio (SMS/WA + webhook) · LinkedIn multi-engine (from C)

Merge plan (in order)

  1. Theme: from B → apply maroon #6B1F2B / gold #C3A35E / cream #F5F1E8 as the active --primary (replace A's teal) + Dancing Script + Inter fonts.
  2. Discovery upgrades: from C → port lib/discover/query-parser (count + company-vs-people intent) and lib/linkedin/search (Serper → Google-HTML → Brave failover, UA rotation, in-memory cache) + edge middleware.js auth pattern.
  3. Scoring + Brand Engine fallback: from D → reuse LeadScoringEngine country/city normalisation map; mount the 4 static templates as the no-key fallback for Brand Engine.
  4. Sequences worker + intelligence: from E → port worker/advance-sequences cron, intelligence/company + intelligence/market, outreach/sequences + outreach/templates.
  5. Voice widget: from F → embed Harvoice as a /voice route + floating widget, repointed from JSON file to the same Prisma DB.
  6. Archive: drop standalone B / C / D / E / Express F. Single repo, single port (3000), single auth, single DB.
